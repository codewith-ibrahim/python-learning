# requirements.txt Kya Hai? — Complete Explanation

# requirements.txt ek SIMPLE TEXT FILE hai jisme
# aapke project ki SAARI DEPENDENCIES (libraries)
# likhi hoti hain. Ye file batati hai ke project
# ko chalane ke liye KAUNSI libraries chahiye.


# ═══════════════════════════════════════════════
# PROBLEM: requirements.txt Ke Bina
# ═══════════════════════════════════════════════

# Socho aapne project banaya, GitHub par upload kiya.
# Doosra banda aapka project clone karega.

# Uske paas libraries NAHI hongi!
# ❌ import requests → ModuleNotFoundError!
# ❌ import flask → ModuleNotFoundError!
# ❌ import django → ModuleNotFoundError!

# Ab wo kaise jaane ke KAUNSI libraries install karni hain?
# Har error aane par ek-ek karke install karega? MUSHKIL!


# ═══════════════════════════════════════════════
# SOLUTION: requirements.txt
# ═══════════════════════════════════════════════

# requirements.txt mein saari libraries likh do.
# Doosra banda EK COMMAND se sab install kar lega!


# ═══════════════════════════════════════════════
# AAPKA requirements.txt EXAMPLE
# ═══════════════════════════════════════════════

# Aapki requirements.txt:
# ┌──────────────────────────┐
# │ requests                 │
# │ python-dotenv            │
# └──────────────────────────┘

# Matlab:
# requests → API calls ke liye
# python-dotenv → .env file load karne ke liye


# ═══════════════════════════════════════════════
# REQUIREMENTS.TXT KAISE BANAYE?
# ═══════════════════════════════════════════════

# Method 1: MANUALLY
# requirements.txt file banao
# Library names likho (Ek line mein ek library)

# Method 2: AUTOMATICALLY
# Terminal mein (Virtual Environment ACTIVE hone ke baad):
# pip freeze > requirements.txt

# pip freeze = Saari installed libraries dikhao
# > requirements.txt = Output ko file mein save karo

# AUTOMATIC OUTPUT:
# requests==2.28.0
# python-dotenv==0.21.0


# ═══════════════════════════════════════════════
# REQUIREMENTS.TXT KAISE USE KARE?
# ═══════════════════════════════════════════════

# Doosra developer jab aapka project clone karega:

# Step 1: Virtual Environment create + activate
# python -m venv venv
# venv\Scripts\activate

# Step 2: requirements.txt se sab install
# pip install -r requirements.txt

# BAS! Saari libraries install ho gayin!


# ═══════════════════════════════════════════════
# VERSIONS KE SAATH vs BINA VERSIONS
# ═══════════════════════════════════════════════

# Bina Version:
# requests
# → Latest version install hogi (Kabhi problem kar sakti hai)

# Version Ke Saath (RECOMMENDED):
# requests==2.28.0
# → Exactly yehi version install hogi (Safe!)

# Minimum Version:
# requests>=2.28.0
# → Ye version ya isse upar wali koi bhi


# ═══════════════════════════════════════════════
# EXAMPLE: BADE PROJECT KI requirements.txt
# ═══════════════════════════════════════════════

# Django==4.2.0
# djangorestframework==3.14.0
# requests==2.28.0
# python-dotenv==0.21.0
# Pillow==9.4.0
# gunicorn==20.1.0
# psycopg2-binary==2.9.5

# Ek command se sab install:
# pip install -r requirements.txt


# ═══════════════════════════════════════════════
# COMMON MISTAKES
# ═══════════════════════════════════════════════

# Mistake 1: requirements.txt ko update karna bhoolna
# Nayi library install ki, requirements.txt update nahi ki!
# Solution: pip freeze > requirements.txt regularly chalao

# Mistake 2: Virtual environment ke bina install karna
# Global Python mein libraries install ho jayengi!
# Solution: Pehle venv activate karo, phir install karo

# Mistake 3: requirements.txt ko GitHub par upload na karna
# Doosre developer ko pata nahi chalega kaunsi libraries chahiye!
# Solution: requirements.txt GitHub par zaroor upload karo!


# ═══════════════════════════════════════════════
# REAL LIFE ANALOGY
# ═══════════════════════════════════════════════

# Socho aap cake banana chahte hain:

# RECIPE (requirements.txt):
# 🥚 Eggs - 2
# 🥛 Milk - 1 cup
# 🍚 Flour - 2 cups
# 🍬 Sugar - 1 cup

# Ab koi bhi is recipe ko follow karke
# same cake bana sakta hai!

# Waise hi:
# requirements.txt = Project ki RECIPE
# pip install -r requirements.txt = Recipe follow karo!
